1. 팀별, 개인별 OOP 적용 범위 항목 리스트 를 확인 후, 선정, 구현 목표 설정, 목표 관리

  1.1 구현 목표의 기준
      - 개인별로 오늘 7교시 까지 마무리 : 1차 버전
      - 개인별로 오늘 중으로 마무리 할 수 있는 범위 : 2차 버전
      - 개인별로 내일(토요일)까 마무리 할 수 있는 범위 : 3차 버전

  1.2 구현 목표 리스트 엑셀 파일 작성.
      - 관리 항목
        클래스 명, 메소드 명, 기능명, 적용 항목 번호, 난이도(상중하), 담당자, 완료 유무, 비고
        
      - 상중하 구분 방법
        상 : 7교 이내로 할 수 없는 어려운 수준.
        중 : 잘하는 팀원에게 도움을 받아서 구현할 수 있는 수준.
            단 오늘 7교시 이내로 완료 할 수 있는것.
        하 : 개인이 설정한 목표가 혼자서 구현을 완료할 수 있는 수준.
            단 오늘 7교시 이내로 완료할 수 있는것.

******** OOP 적용 범위 항목 리스트 ********

1. 캡슐화
   접근제어자
   
2. 상속
   2.1 상속 관계 결정
       is-a관계(상속관계)
       has-a관계(포함관계)
   2.2 오버라이딩
   2.3 Object class 오버라이딩
   
3. 추상화
   3.1 추상클래스
   3.2 인터페이스
   
4. 다형성
   4.1 매개변수의 다형성 : 상속
   4.2 반환타입의 다형성 : 인터페이스
   4.3 객체관리 : 객체배열 : 상속(책기준), 인터페이스
   4.4 객체관리 : Vector : 상속(책기준), 인터페이스
   4.5 인터페이스를 활용한 다형성
   
5. 유지보수성(약한 결합)
   인터페이스 활용
   
6. 클래스 디자인
   6.1 package 구성 - controller, entity
   6.2 추상화( 추상클래스, 인터페이스 )
   
7. 실행
   .jar 파일로 패키징을 한 후 java 명령어로 실행
